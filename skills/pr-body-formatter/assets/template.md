# Pull Request Body Template

## Summary

Write one concise paragraph that names the user-visible behavior and the issue being closed.

## Changes

- Name each changed file or module and the behavior added there.
- Group tiny mechanical edits together.
- Mention any migration, compatibility, or follow-up risk.

## How tested

- List exact commands run and their result.
- Include manual verification only when it adds information not covered by automated tests.

## Test plan

- [ ] CI green
- [ ] Maintainer-specific acceptance check, if any
