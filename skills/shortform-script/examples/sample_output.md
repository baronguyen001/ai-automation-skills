## Script

Hook: Your first CLI does not need a plugin system. It needs one command that works.

Beat 1: Start with the user action. "I want to lint one folder and see the errors."

Beat 2: Ship the smallest command that proves the value: `tool check ./project`.

Beat 3: Do not add config files, plugin loading, or themes until a real user repeats the same request.

Beat 4: Keep one proof slot for your own project: [NUMBER: measured setup time before and after simplification].

CTA: Save this before your next "quick" CLI turns into a framework.
